This document records all major updates to the Cobo payments solution.

## October 2025

| Product | Change description | Release date |
|------|----------|----------|
| **Payments API** |To simplify the settlement process and improve user experience, we have optimized Payments API by removing concepts and related operations such as withdrawable balance and forced sweep:<br/><br/>**Deprecated operations:**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- Withdrawable balance query: [Get withdrawable balances](/v2/api-references/payment/get-withdrawable-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Payment wallet balance query: [List payment wallet balances](/v2/api-references/payment/list-payment-wallet-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Forced sweep operation: [Create forced sweep](/v2/api-references/payment/create-forced-sweep), [List forced sweeps](/v2/api-references/payment/list-forced-sweeps)<br/><br/>**Deprecated fields:**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- `available_balance` field in [List merchant balances](/v2/api-references/payment/list-merchant-balances) and [Get developer balance](/v2/api-references/payment/get-developer-balance)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `use_dedicated_address` field in [Create pay-in order](/v2/api-references/payment/create-pay-in-order)<br/><br/><Warning>The above operations and fields are retained only for compatibility with older versions and should not be used in new implementations.</Warning> | 2025-10-01 |

## August 2025

| Product | Change description | Release date |
|------|----------|----------|
| **Payments API** |API SDK v1.22.0 released. Changes as follows:<br/><br/>- **Breaking change**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;-The ​`settlements.token_id` field in the [Create settlement request](/v2/api-references/payment/create-settlement-request) request body is changed from optional to required. At the same time, the ​`settlements.amount` field only represents the cryptocurrency amount to be settled and no longer represents the fiat currency amount.<br/><br/>- Added balance-related operations:<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List merchant balances](/v2/api-references/payment/list-merchant-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Get developer balance](/v2/api-references/payment/get-developer-balance)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List payment wallet balances](/v2/api-references/payment/list-payment-wallet-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Get payer balance](/v2/api-references/payment/get-payer-balance)| 2025-08-28 |
| **Payments API** |API SDK v1.21.0 released. Changes as follows:<br/><br/>- Added settlement-related operations:<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List all settlement details](/v2/api-references/payment/list-all-settlement-details): Used to query settlement details.<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create crypto address](/v2/api-references/payment/create-crypto-address): Used to register crypto payout addresses<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Delete crypto address](/v2/api-references/payment/delete-crypto-address): Used to cancel registration of crypto payout addresses<br/><br/>- Added refund-related operations:<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Update refund order](/v2/api-references/payment/update-refund-order): Used to update the receiving address of refund orders.<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Cancel refund order](/v2/api-references/payment/cancel-refund-order): Used to cancel refund orders| 2025-08-13 |
| **Checkout SDK** | - Added support for payment through third-party wallets, supporting mainstream wallets on EVM and Tron chains.<br/>- Added support for directly specifying order amount in cryptocurrency through the `cryptoAmount` parameter.| 2025-08-11 |


## July 2025

| Product | Change description | Release date |
|------|----------|----------|
| **Checkout SDK** | Added support for customizing the "Powered by xxx" text at the bottom of the checkout page through the `developerName` parameter. | 2025-07-30 |
| **Payments API** |API SDK v1.20.0 released. Changes as follows:<br/><br/>- [Create pay-in order](/v2/api-references/payment/create-pay-in-order) new feature: Supports directly specifying order amount in cryptocurrency. When the `currency` parameter is empty, `order_amount` and `fee_amount` will be in the cryptocurrency specified by `token_id` instead of fiat currency.<br/><br/>- Added [Update top-up address](/v2/api-references/payment/update-top-up-address) operation to update top-up addresses.<br/><br/> - Added forced sweep-related operations:<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create forced sweep](/v2/api-references/payment/create-forced-sweep)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List forced sweeps](/v2/api-references/payment/list-forced-sweeps)<br/><br/>- [Get withdrawable balances](/v2/api-references/payment/get-withdrawable-balances) added `available_balance` and `total_balance` fields, representing withdrawable balance and total balance respectively.<br/><br/>- Added `payment.address.updated` webhook event to notify top-up address updates. At the same time, order-related events have added multiple fields.| 2025-07-30 |
| **Payments API** | - [Create settlement request](/v2/api-references/payment/create-settlement-request) new feature: When the `order_ids` parameter is empty, automatically settle all unsettled orders for the merchant. | 2025-07-24 |
| **Payments App** | - Added support for top-up mode. You can now on the web interface:<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Create and manage top-up addresses<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Monitor top-up transaction status<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Export top-up records | 2025-07-10 |
| **Payments API** | API SDK v1.18.0 released. Changes as follows:<br/><br/>- Added top-up mode-related operations:<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Get top-up address](/v2/api-references/payment/get-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List payers](/v2/api-references/payment/list-payers)<br/>&nbsp;&nbsp;For specific integration steps, refer to [Get top-up address](/v2/payments/get-topup-addresses)<br/><br/>- Added `use_dedicated_address` parameter to [Create pay-in order](/v2/api-references/payment/create-pay-in-order), supporting specifying the use of a dedicated receiving address<br/><br/>- Added multiple [webhook events](/v2/payments/status-and-events) related to top-up mode | 2025-07-02 |
