## Producer scope

`wallets.mpc.tss_request.*` events are emitted only for TSS requests of type `KeyGen`, `KeyGenFromKeyGroup`, or `Recovery`. These types cover Main Group key generation, Signing or Recovery Group provisioning from an existing Main Group, and lost-key-share recovery. Routine MPC Wallet transaction signing does not create a TSS request. Follow that activity through the signed transaction's `wallets.transaction.*` events instead.

## Triggers

| Event type | When it occurs |
| --- | --- |
| `wallets.mpc.tss_request.created` | A TSS request is created. |
| `wallets.mpc.tss_request.updated` | The request enters `PendingKeyHolderConfirmation`. The `KeyGenerating` and `MPCProcessing` intermediate states do not dispatch a webhook. |
| `wallets.mpc.tss_request.succeeded` | The requested action has been successfully completed. |
| `wallets.mpc.tss_request.failed` | The requested action failed. |

### State-to-event mapping

| State or transition | Event |
| --- | --- |
| Request creation | `wallets.mpc.tss_request.created` |
| `PendingKeyHolderConfirmation` | `wallets.mpc.tss_request.updated` |
| `Success` | `wallets.mpc.tss_request.succeeded` |
| `KeyHolderConfirmationFailed` or `KeyGeneratingFailed` | `wallets.mpc.tss_request.failed` |

Transitions to `KeyGenerating` and `MPCProcessing` are suppressed and do not dispatch any `wallets.mpc.tss_request.*` event.

## Webhook envelope

For these events, `data.data_type` is always `TSSRequest`.

| Field | Description |
| --- | --- |
| `event_id` | UUID of this delivery record. Use it to deduplicate retries of the same event. |
| `url` | Endpoint URL to which the event was sent. |
| `created_timestamp` | Time the event was triggered, as a Unix timestamp in milliseconds. This value does not change across retries. |
| `type` | Webhook event type. |
| `data` | Object containing `data_type` and the TSS request payload in `data`. |
| `status` | Delivery status: `Success`, `Retrying`, or `Failed`. This is separate from the TSS request status. |
| `next_retry_timestamp` | Next retry time in milliseconds. Present only when `status` is `Retrying`. |
| `retries_left` | Remaining delivery retries. Present only when `status` is `Retrying`. |

### `source_key_share_holder_group`

When `source_key_share_holder_group` is not `null`, it contains:

| Field | Type | Description |
| --- | --- | --- |
| `key_share_holder_group_id` | string | Unique identifier of the existing key share holder group. |
| `name` | string | Name of the existing key share holder group. |

## Statuses

| Status | Meaning |
| --- | --- |
| `PendingKeyHolderConfirmation` | Waiting for enough key share holders to approve the request. |
| `KeyHolderConfirmationFailed` | Key share holders failed to approve the request. |
| `KeyGenerating` | Key shares are being generated. |
| `MPCProcessing` | Approval or signing is in progress through a server or mobile co-signer. See [Introduction to server co-signers](/v2/guides/mpc-wallets/server-co-signer/tss-node-overview). |
| `KeyGeneratingFailed` | Key-share generation failed. |
| `Success` | The action completed, including a successful cancellation. |
