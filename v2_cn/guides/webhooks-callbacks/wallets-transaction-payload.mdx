## 触发条件

| 事件类型 | 发生时机 |
| --- | --- |
| `wallets.transaction.created` | 已创建交易记录。对于充币等入账交易，会在链上检测到交易后创建记录；对于提币等出账交易，会在 Cobo 根据请求创建记录后触发。此事件不代表交易已成功。 |
| `wallets.transaction.updated` | 交易的状态或确认数发生变化。 |
| `wallets.transaction.succeeded` | 交易已成功完成。 |
| `wallets.transaction.failed` | 交易失败。 |

这些事件适用于所有交易类型，包括充币、提币、合约调用、消息签名、质押和解除质押。

### 状态到事件的映射

| Transaction 状态 | 额外的终态事件 |
| --- | --- |
| `Completed` | `wallets.transaction.succeeded` |
| `Failed`、`Rejected` 或 `Canceled` | `wallets.transaction.failed` |

除 `wallets.transaction.updated` 外，还会尝试触发对应的终态事件。有关事件类型如何重叠，请参阅[单笔交易的事件重叠](/v2_cn/guides/webhooks-callbacks/webhook-event-type#单笔交易的事件重叠)。

## Webhook 信封

每次投递都包含以下信封。对于这些事件，`data.data_type` 始终为 `Transaction`。

| 字段 | 描述 |
| --- | --- |
| `event_id` | 本次投递记录的 UUID，用于对同一事件的重试投递进行去重。 |
| `url` | 事件投递到的 Endpoint URL。 |
| `created_timestamp` | 事件触发时间，采用毫秒级 Unix 时间戳。重试时此值不变。 |
| `type` | Webhook 事件类型。 |
| `data` | 包含 `data_type` 标识以及位于 `data` 中的交易负载。 |
| `status` | 此 Endpoint 的投递状态：`Success`、`Retrying` 或 `Failed`。此字段不同于交易业务状态。 |
| `next_retry_timestamp` | 下次重试时间，采用毫秒级 Unix 时间戳。仅在 `status` 为 `Retrying` 时出现。 |
| `retries_left` | 剩余投递重试次数。仅在 `status` 为 `Retrying` 时出现。 |

### `raw_tx_info`

| 字段 | 描述 |
| --- | --- |
| `used_nonce` | 交易使用的 Nonce。 |
| `raw_tx` | 序列化的已签名交易。 |
| `unsigned_raw_tx` | 序列化的未签名交易。 |
| `selected_utxos` | UTXO 链交易消耗的输入。 |
| `utxo_changes` | UTXO 链交易产生的找零输出。 |

### `replacement`

| 字段 | 描述 |
| --- | --- |
| `replaced_by_type` | 对原交易执行的替换操作。 |
| `replaced_by_transaction_id` | 替代原交易的新交易 ID。 |
| `replaced_by_transaction_hash` | 替代原交易的新交易哈希。 |
| `replaced_type` | 创建新交易的替换操作。 |
| `replaced_transaction_id` | 原交易 ID。 |
| `replaced_transaction_hash` | 原交易哈希。 |

有关完整替换流程，请参阅 [RBF Transaction 生命周期](/v2_cn/guides/transactions/rbf-transaction-lifecycle)；有关 `ReplacedByNewTransaction`，请参阅 [Transaction 状态和子状态](/v2_cn/guides/transactions/status)。

### `timeline`

每个 `timeline` 条目包含：

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| `status` | string | 此生命周期阶段所表示的 Transaction 状态。 |
| `finished` | boolean | 交易是否已完成此生命周期阶段。 |
| `finished_timestamp` | integer，可为 null | 此阶段的完成时间，采用毫秒级 Unix 时间戳。阶段尚未完成时为 `null`。 |

## 字段语义

`data.data` 对象包含上述交易字段。有关 `token_id` 和 `asset_id` 的区别，请参阅 [Transaction 字段](#transaction-字段)。

有关 `cobo_id` 和 `request_id` 在交易生命周期中的可用性，请参阅 [Transaction 字段](#transaction-字段)中的对应条目。`result` 仅用于已完成的 `MessageSign` 交易。`source` 和 `destination` 结构因交易类型而异；请参阅 [Transaction 来源和目标](/v2_cn/guides/transactions/sources-and-destinations)。

替换字段使用相同的 Transaction 结构，并关联原交易和替换交易。同样，当 `cobo_category` 包含 `AutoSweep` 时，该 AutoSweep 交易也使用与其他提币相同的 Transaction 结构。
