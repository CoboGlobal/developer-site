## 产生范围

仅 `KeyGen`、`KeyGenFromKeyGroup` 或 `Recovery` 类型的 TSS request 会产生 `wallets.mpc.tss_request.*` 事件。这些类型涵盖 Main Group 私钥分片生成、基于现有 Main Group 创建 Signing Group 或 Recovery Group，以及丢失私钥分片后的恢复。MPC Wallet 的常规交易签名不会创建 TSS request。请通过已签名交易的 `wallets.transaction.*` 事件跟踪此类活动。

## 触发条件

| 事件类型 | 发生时机 |
| --- | --- |
| `wallets.mpc.tss_request.created` | 创建了 TSS request。 |
| `wallets.mpc.tss_request.updated` | 请求进入 `PendingKeyHolderConfirmation`。`KeyGenerating` 和 `MPCProcessing` 中间状态不会派发 Webhook。 |
| `wallets.mpc.tss_request.succeeded` | 请求的操作已成功完成。 |
| `wallets.mpc.tss_request.failed` | 请求的操作失败。 |

### 状态到事件的映射

| 状态或转换 | 事件 |
| --- | --- |
| 创建请求 | `wallets.mpc.tss_request.created` |
| `PendingKeyHolderConfirmation` | `wallets.mpc.tss_request.updated` |
| `Success` | `wallets.mpc.tss_request.succeeded` |
| `KeyHolderConfirmationFailed` 或 `KeyGeneratingFailed` | `wallets.mpc.tss_request.failed` |

转换到 `KeyGenerating` 和 `MPCProcessing` 时会抑制事件，不会派发任何 `wallets.mpc.tss_request.*` 事件。

## Webhook 信封

对于这些事件，`data.data_type` 始终为 `TSSRequest`。

| 字段 | 描述 |
| --- | --- |
| `event_id` | 本次投递记录的 UUID，用于对同一事件的重试投递进行去重。 |
| `url` | 事件投递到的 Endpoint URL。 |
| `created_timestamp` | 事件触发时间，采用毫秒级 Unix 时间戳。重试时此值不变。 |
| `type` | Webhook 事件类型。 |
| `data` | 包含 `data_type` 以及位于 `data` 中的 TSS request 负载。 |
| `status` | 投递状态：`Success`、`Retrying` 或 `Failed`。此字段不同于 TSS request 状态。 |
| `next_retry_timestamp` | 下次重试时间。仅在 `status` 为 `Retrying` 时出现。 |
| `retries_left` | 剩余投递重试次数。仅在 `status` 为 `Retrying` 时出现。 |

### `source_key_share_holder_group`

当 `source_key_share_holder_group` 不为 `null` 时，包含：

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| `key_share_holder_group_id` | string | 现有 key share holder group 的唯一标识。 |
| `name` | string | 现有 key share holder group 的名称。 |

## 状态

| 状态 | 含义 |
| --- | --- |
| `PendingKeyHolderConfirmation` | 等待足够数量的私钥分片持有者批准请求。 |
| `KeyHolderConfirmationFailed` | 私钥分片持有者未能批准请求。 |
| `KeyGenerating` | 正在生成私钥分片。 |
| `MPCProcessing` | 正在通过 server co-signer 或 mobile co-signer 进行审批或签名。请参阅 [server co-signer 介绍](/v2_cn/guides/mpc-wallets/server-co-signer/tss-node-overview)。 |
| `KeyGeneratingFailed` | 私钥分片生成失败。 |
| `Success` | 操作已完成，包括成功取消。 |
